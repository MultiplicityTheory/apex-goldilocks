# ADR-034: Twisted Edwards as Circuit Optimization Layer

**Status:** Deferred  
**Date:** 2026-05-16  
**Owners:** Circuit Design Layer  

## Context
To improve ZK-circuit performance and reduce constraint counts for EC-heavy operations (e.g., Pedersen commitments, signature verification), a more efficient coordinate representation may be needed.

## Decision
Twisted Edwards coordinate forms for supported curves (e.g., Pallas/Vesta) are designated as an optional circuit-level optimization, to be explored only after baseline Weierstrass-form proof generation is validated.

## Compliance with Sedona Spine
- **Coordinate Model**: Does not change the field arithmetic, only the coordinate model within ZK constraints.
- **Consistency**: Must be implemented within the field-native framework.

## Consequences
- **Performance**: Potential reduction in circuit constraint count.
- **Complexity**: Adds coordinate transformation logic within ZK-circuits.
