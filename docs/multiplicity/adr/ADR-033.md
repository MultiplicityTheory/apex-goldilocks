# ADR-033: Pallas/Vesta as Optional EC Commitment Layer

**Status:** Deferred  
**Date:** 2026-05-16  
**Owners:** Cryptographic Layer  

## Context
Future requirements may include privacy-preserving governance votes, zero-knowledge membership proofs, or recursive proof composition using elliptic curve commitments.

## Decision
Pallas and Vesta are designated as the optional Elliptic Curve (EC) commitment layer. They are strictly deferred for proof-time-only operations and must NOT be used in the hot runtime kernel path.

## Compliance with Sedona Spine
- **Isolation**: EC arithmetic (254-bit) is isolated from the 64-bit Goldilocks runtime path.
- **No Runtime Impact**: EC layer remains a "cold path" component.

## Consequences
- **Security**: Provides a solid basis for future ZK-governance and proof recursion.
- **Performance**: High-cost EC arithmetic is strictly separated from time-critical spectral updates.
