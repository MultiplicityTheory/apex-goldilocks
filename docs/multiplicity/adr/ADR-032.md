# ADR-032: Goldilocks Kernel as Canonical Runtime Field

**Status:** Accepted  
**Date:** 2026-05-16  
**Owners:** Core Arithmetic Layer  

## Context
The system requires a high-throughput, FFT-friendly field for runtime spectral state management and ZK-proof generation. Plonky3 provides first-class support for the Goldilocks field (\(\mathbb{F}_{\mathrm{Gold}}\), \(p = 2^{64} - 2^{32} + 1\)), which is naturally suited for high-performance arithmetic on 64-bit hardware.

## Decision
Adopt the Goldilocks field (\(\mathbb{F}_{\mathrm{Gold}}\)) as the canonical arithmetic base for all runtime state, spectral quantities, resonance amplitudes, and ZK-proof public inputs.

## Compliance with Sedona Spine
- **Zero Drift**: Provides an exact-arithmetic foundation.
- **No Floats**: Replaces all floating-point operations with exact 64-bit integer arithmetic.
- **Rust-Native**: Leverages native `u64` arithmetic and optimized SIMD kernels.

## Consequences
- **Performance**: High throughput for spectral operations via SIMD.
- **Reproducibility**: Canonical field ensures identical results across all hardware/compilers.
- **Ecosystem**: Aligns with Plonky3's proven, modular proving stack.
